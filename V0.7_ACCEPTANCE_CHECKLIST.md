# ResumeFit V0.7 Acceptance Checklist

本文档用于验收 ResumeFit V0.7：DOCX 简历导出增强版。

## 一、V0.7 已完成内容

- 后端新增 `GET /resume-versions/{id}/export/docx`。
- 使用 `python-docx` 基于 `ResumeVersion.content_markdown` 生成 DOCX。
- DOCX 导出以内存流返回，不长期保存到公开目录。
- DOCX 导出不调用 AI。
- DOCX 导出不计入 AI 使用额度。
- DOCX 导出沿用 V0.3 用户隔离，用户只能导出自己的 `ResumeVersion`。
- 前端 `/versions` 页面新增“导出 DOCX”按钮。
- DOCX 下载请求携带 `Authorization: Bearer <token>`。
- DOCX 下载支持解析 `Content-Disposition` 文件名。
- Markdown 导出继续保留。
- PDF 导出暂未实现，留作后续版本规划。

## 二、手动验收步骤

### 1. Git 状态检查

- [ ] 操作：在项目根目录执行 `git status`。
- [ ] 预期：没有 `.env`、SQLite 数据库、`.docx` 导出文件、临时日志或构建缓存等待提交。

### 2. 后端测试

- [ ] 操作：
  ```powershell
  cd backend
  .\.venv\Scripts\python.exe -m pytest
  ```
- [ ] 预期：测试通过，当前验收结果为 `88 passed, 3 warnings`。

### 3. 前端构建

- [ ] 操作：
  ```powershell
  cd frontend
  npm run build
  ```
- [ ] 预期：构建通过。
- [ ] 注意：如 `frontend/tsconfig.tsbuildinfo` 出现变更，提交前还原。

### 4. DOCX 后端接口验收

- [ ] 未登录访问 `GET /resume-versions/{id}/export/docx` 返回 `401`。
- [ ] 用户 A 可以导出自己的 DOCX。
- [ ] 用户 B 导出用户 A 的 ResumeVersion 返回 `404`。
- [ ] 不存在的 ResumeVersion 返回 `404`。
- [ ] 响应 `Content-Type` 为 `application/vnd.openxmlformats-officedocument.wordprocessingml.document`。
- [ ] `Content-Disposition` 包含 `.docx` 安全文件名。
- [ ] 文件名不包含 token、email、用户 id 等敏感信息。
- [ ] DOCX 内容来自 `ResumeVersion.content_markdown`。
- [ ] DOCX 内容包含标题、技能或项目段落等核心文本。
- [ ] DOCX 导出不调用 AI。
- [ ] DOCX 导出不计入 AI 使用额度。

### 5. `/versions` 页面 DOCX 下载验收

- [ ] 未登录访问 `/versions` 会跳转 `/login`。
- [ ] 登录后选择一个已生成的 ResumeVersion。
- [ ] 点击“导出 DOCX”。
- [ ] 预期：浏览器下载 `.docx` 文件。
- [ ] 预期：下载请求携带 `Authorization`，但 token 不出现在 URL 中。
- [ ] 预期：导出中按钮显示 loading，并防止重复点击。
- [ ] 预期：失败时显示友好错误提示。
