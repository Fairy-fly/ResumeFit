# ResumeFit Demo 手动验收检查表

## 0. 验收前准备

### 启动后端

1. 打开 PowerShell。
2. 执行：

```powershell
cd backend
.\.venv\Scripts\Activate.ps1
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

预期结果：

- 终端显示 Uvicorn 正在运行。
- 访问 `http://localhost:8000/health` 返回 `{"status":"ok"}`。

### 启动前端

1. 新开一个 PowerShell。
2. 执行：

```powershell
cd frontend
npm run dev
```

预期结果：

- 终端显示 Vite 本地地址。
- 浏览器可以访问 `http://localhost:5173`。

### AI 配置检查

1. 确认项目根目录存在 `.env`。
2. 确认 `.env` 中配置了：

```text
AI_PROVIDER=deepseek
AI_BASE_URL=https://api.deepseek.com
AI_API_KEY=your_deepseek_api_key_here
AI_MODEL=deepseek-chat
```

预期结果：

- 如果配置了真实本地 API Key，AI 模块可以正常调用。
- 如果没有配置真实 API Key，AI 相关页面应显示明确错误提示，不应导致页面崩溃。

## 10. 常见错误排查

### 10.1 前端打不开

现象：

- `http://localhost:5173` 无法访问。

排查：

1. 确认 `npm run dev` 是否正在运行。
2. 确认终端中显示的端口是否为 `5173`。
3. 如果端口变化，以 Vite 终端输出的地址为准。

### 10.2 后端接口失败

现象：

- 页面提示请求失败。
- `http://localhost:8000/health` 无法访问。

排查：

1. 确认后端 `uvicorn` 是否正在运行。
2. 确认后端端口是否为 `8000`。
3. 查看后端终端错误日志。

### 10.3 AI 调用失败

现象：

- 页面提示 `AI_API_KEY is not configured.`
- 页面提示 AI 返回非 JSON 或网络错误。

排查：

1. 检查 `.env` 是否存在。
2. 检查 `AI_API_KEY` 是否填写本地真实值。
3. 检查 `AI_BASE_URL` 和 `AI_MODEL` 是否正确。
4. 检查网络是否能访问模型服务。
5. 如果现场无法调用 AI，可展示历史结果、Prompt 文件和 pytest mock AI 通过截图。

### 10.4 页面数据为空

现象：

- `/analysis` 或 `/versions` 没有可选数据。

排查：

1. 先在 `/resume` 保存简历。
2. 再在 `/projects` 保存项目。
3. 再在 `/jobs` 保存并分析 JD。
4. 再去 `/analysis` 生成匹配报告。
5. 最后去 `/versions` 生成定制简历。

### 10.5 SQLite 结构异常

现象：

- 后端提示某个字段不存在。
- 旧数据导致页面异常。

排查：

1. 停止后端。
2. Demo 阶段可删除本地数据库：

```powershell
Remove-Item backend/resumefit.db -ErrorAction SilentlyContinue
```

3. 重新启动后端，让应用重新创建表。
