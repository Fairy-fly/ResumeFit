# ResumeFit V0.5 个人中心与账户设置验收清单

本文档用于验收 ResumeFit V0.5。V0.5 只实现个人中心、账户信息展示、昵称修改和用量概览，不包含支付、会员套餐、订单、管理员后台、密码修改、邮箱验证、找回密码或头像上传。

## 一、V0.5 已完成内容

- [ ] 新增 `/account` 个人中心页面。
- [ ] 侧边栏新增“个人中心”入口。
- [ ] 新增 `GET /account/me`。
- [ ] 新增 `PATCH /account/me`。
- [ ] `GET /account/me` 返回当前用户账户信息和 `usage_summary`。
- [ ] 个人中心展示 `email`、`display_name`、`status`、`created_at`、`updated_at`。
- [ ] 个人中心展示 V0.4 用量概览。
- [ ] 支持修改当前用户自己的 `display_name`。
- [ ] 修改昵称后页面和 Sidebar 同步更新。
- [ ] 提供跳转 `/usage` 查看详细用量的入口。
- [ ] 商业化入口仅为静态预留说明，不包含支付、会员套餐或订单。

## 二、后端接口验收

### 1. 未登录访问账户接口

操作步骤：

1. 不携带 token 请求 `GET /account/me`。
2. 不携带 token 请求 `PATCH /account/me`。

预期结果：

- [ ] `GET /account/me` 返回 `401`。
- [ ] `PATCH /account/me` 返回 `401`。

### 2. 登录后读取账户信息

操作步骤：

1. 登录任意测试用户。
2. 携带 `Authorization: Bearer <token>` 请求 `GET /account/me`。

预期结果：

- [ ] 返回当前用户 `id`。
- [ ] 返回当前用户 `email`。
- [ ] 返回当前用户 `display_name`。
- [ ] 返回当前用户 `status`。
- [ ] 返回 `created_at`。
- [ ] 返回 `updated_at`。
- [ ] 返回 `usage_summary`。

### 3. 修改昵称

操作步骤：

1. 登录用户 A。
2. 调用 `PATCH /account/me`：

```json
{
  "display_name": "User A New Name"
}
```

3. 再调用 `GET /account/me`。
4. 再调用 `GET /auth/me`。

预期结果：

- [ ] `PATCH /account/me` 返回新昵称。
- [ ] `GET /account/me` 返回新昵称。
- [ ] `GET /auth/me` 返回新昵称。
- [ ] 只修改 `display_name`，不修改 `email`、`status`、`password_hash`、角色或额度字段。

### 4. 昵称校验

操作步骤：

1. 用空字符串调用 `PATCH /account/me`。
2. 用全空白字符串调用 `PATCH /account/me`。
3. 用超过 50 个字符的字符串调用 `PATCH /account/me`。

预期结果：

- [ ] 空昵称返回 `422`。
- [ ] 全空白昵称返回 `422`。
- [ ] 超过 50 个字符返回 `422`。

### 5. 用户隔离

操作步骤：

1. 登录用户 A 并修改昵称。
2. 登出用户 A。
3. 登录用户 B。
4. 打开 `/account` 或调用 `GET /account/me`。

预期结果：

- [ ] 用户 B 只能看到自己的账户信息。
- [ ] 用户 B 不受用户 A 昵称修改影响。
