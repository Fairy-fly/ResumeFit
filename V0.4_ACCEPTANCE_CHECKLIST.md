# ResumeFit V0.4 AI 用量与基础额度验收清单

本文档用于验收 ResumeFit V0.4。V0.4 只实现 AI 调用日志、用量统计、基础月额度和 `/usage` 页面，不包含支付、会员套餐、订单、管理员后台或发票。

## 一、V0.4 已完成内容

- [ ] 新增 `ai_usage_logs` 表。
- [ ] AI 调用日志按 `user_id` 隔离。
- [ ] 日志记录功能类型：`jd_analysis`、`match_report`、`resume_generation`、`truth_check`、`interview_question`。
- [ ] 日志记录模型名称、成功/失败、错误信息和创建时间。
- [ ] token usage 字段允许为空。
- [ ] `estimated_cost` 字段已预留。
- [ ] `.env` 支持 `AI_MONTHLY_CALL_LIMIT`。
- [ ] 超过月额度时返回 `429 Monthly AI quota exceeded.`。
- [ ] 新增 `GET /usage/summary`。
- [ ] 前端新增 `/usage` 用量统计页面。
- [ ] 侧边栏新增“用量统计”入口。

## 二、手动验收步骤

### 1. 空账号用量页

操作步骤：

- 注册或登录一个新账号。
- 打开 `/usage`。

预期结果：

- [ ] 页面显示本月额度、已用次数、剩余额度。
- [ ] 已用次数为 `0`。
- [ ] 最近调用记录为空。
- [ ] 页面显示空状态提示。

### 2. 成功 AI 调用日志

操作步骤：

- 进入 `/jobs`。
- 创建一个 JD 并点击“保存并分析”。
- 返回 `/usage`。

预期结果：

- [ ] 本月已用次数增加。
- [ ] 各功能调用次数中出现 `JD 分析`。
- [ ] 最近调用记录出现一条成功记录。
- [ ] 记录中显示模型名称。

### 3. 失败 AI 调用日志

操作步骤：

- 临时移除或置空 `backend/.env` 中的 `AI_API_KEY`。
- 重启后端。
- 触发一次 JD 分析或其他 AI 功能。
- 打开 `/usage`。

预期结果：

- [ ] 页面显示 AI Key 未配置的友好错误。
- [ ] `/usage` 中失败次数增加。
- [ ] 最近调用记录显示失败状态和错误信息。

### 4. 月额度超限

操作步骤：

- 将 `AI_MONTHLY_CALL_LIMIT` 临时设为较小值，例如 `1`。
- 重启后端。
- 完成一次 AI 调用。
- 再触发第二次 AI 调用。

预期结果：

- [ ] 第二次调用返回额度耗尽提示。
- [ ] 前端显示“本月 AI 调用额度已用完”。
- [ ] 第二次调用不会生成新的业务结果。

### 5. 双账号用量隔离

操作步骤：

- 用户 A 触发一次 AI 调用。
- 用户 A 打开 `/usage` 确认有调用记录。
- 退出用户 A，登录用户 B。
- 用户 B 打开 `/usage`。

预期结果：

- [ ] 用户 B 看不到用户 A 的 AI 调用记录。
- [ ] 用户 B 的用量统计从自己的数据开始计算。

## 三、接口级验收

Swagger 地址：

```text
http://localhost:8000/docs
```

需要先通过 `/auth/login` 获取 token，并在 Swagger Authorize 中填写：

```text
Bearer <access_token>
```

接口检查：

- [ ] 无 token 调用 `GET /usage/summary` 返回 `401`。
- [ ] 有效 token 调用 `GET /usage/summary` 返回当前用户统计。
- [ ] 用户 A 和用户 B 的 `/usage/summary` 结果互相隔离。
- [ ] 额度超限时 AI 功能接口返回 `429`。
