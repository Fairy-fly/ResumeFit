# ResumeFit 答辩讲稿

## 1. 开场介绍

各位老师好，我的项目名称是 ResumeFit，这是一个基于 Vue 3、FastAPI、SQLite 和大语言模型 API 的智能简历定制与求职辅助平台。

这个项目的目标不是做一个完整商业产品，而是完成一个可以演示的课程项目 Demo，验证从“用户输入真实简历材料”到“生成岗位定制简历、检测真实性风险、准备面试追问”的完整流程。

## 2. 项目背景说明

在实际求职中，很多同学会同时投递多个岗位，但通常只有一份通用简历。不同岗位 JD 对技能、项目经历和表达重点的要求不同，如果每次都手动修改简历，会比较耗时。

现在也有很多人会直接使用 AI 改简历，但这会带来一个问题：AI 可能为了让简历更匹配岗位，写出用户并没有提供的经历、成果或技能。这样的简历虽然看起来更强，但面试时很容易被追问，甚至影响诚信。

所以我设计 ResumeFit 的核心思路是：基于用户真实提供的信息进行整理、突出和改写，同时检测可能存在的夸大和不确定内容。

## 3. 需求分析说明

项目主要面向准备实习、校招、转岗或社招投递的个人用户。用户的核心需求可以分为四类：

第一，能够保存自己的通用简历和项目经历，形成事实来源。

第二，能够快速理解目标岗位 JD 的要求，比如必备技能、加分技能、岗位职责和关键词。

第三，能够知道自己的简历和岗位之间的匹配程度，并获得修改建议。

第四，能够生成一版针对岗位的 Markdown 简历，同时检查真实性风险，并准备面试中可能被追问的问题。

MVP 阶段不做登录、支付、会员、招聘网站爬取、PDF/Word 导出和企业后台，重点是先跑通核心 Demo。

## 4. 系统功能介绍

当前系统已经完成了 9 个主要功能模块：

1. 首页 Dashboard：展示完整使用流程和当前数据统计。
2. 通用简历输入：用户可以保存简历标题和简历正文。
3. 项目库管理：用户可以保存项目名称、类型、角色、技术栈、项目描述、个人贡献和作品链接。
4. 岗位 JD 分析：用户输入公司名称、岗位名称和 JD 原文，系统调用 AI 输出结构化分析。
5. 匹配度报告：用户选择简历、项目和已分析 JD，系统生成匹配分数、优势、不足和建议。
6. 定制简历生成：系统生成 Markdown 格式的岗位定制简历。
7. 真实性风险检测：系统检查简历中可能夸大、缺证据或不确定的表达。
8. 面试追问预测：系统根据简历、岗位和风险点生成可能被问的问题和建议回答。
9. Markdown 导出：用户可以把生成的简历版本导出为 `.md` 文件。

## 5. 技术架构介绍

项目采用前后端分离架构。

前端使用 Vue 3、TypeScript、Vite 和 Vue Router，主要负责页面展示、表单交互、结果卡片和文件下载。

后端使用 FastAPI，按 route、schema、service、repository、model 分层。route 负责 HTTP 请求，schema 负责请求响应结构，service 编排业务流程，repository 负责数据库读写，model 定义 SQLite 表结构。

数据库使用 SQLite，适合 Demo 阶段快速启动和本地演示。后续如果商业化，可以升级到 PostgreSQL。

AI 调用通过统一的 `AIClient` 实现，使用 OpenAI-compatible 格式。MVP 默认配置 DeepSeek，但代码结构没有和 DeepSeek 强绑定，后续可以切换到 OpenAI、通义、智谱或 OpenRouter 等兼容接口。

## 6. 核心模块讲解

第一个核心模块是 JD 分析。用户粘贴岗位 JD 后，系统将公司名称、岗位名称和 JD 原文发送给后端。后端读取 `prompts/jd_analyzer_v1.md`，调用 AI 并要求返回 JSON，包含岗位类型、必备技能、加分技能、职责、关键词和简历侧重点建议。

第二个核心模块是匹配度报告。系统读取用户选择的通用简历、项目经历、岗位 JD 和 JD 分析结果，再调用 `match_scorer_v1.md`，输出匹配分数、优势、不足、缺失关键词、修改建议和真实性提醒。

第三个核心模块是定制简历生成。系统基于简历、项目、JD 分析和匹配报告，调用 `resume_writer_v1.md`，输出 Markdown 简历，同时输出 `change_explanations`，说明每处修改原因。

第四个核心模块是真实性风险检测。系统读取生成后的 ResumeVersion 和原始材料，调用 `truth_checker_v1.md`，检查是否存在编造经历、技能夸大、无证据量化成果、角色夸大等风险。

第五个核心模块是面试追问预测。系统读取定制简历、岗位信息、项目经历、匹配报告和可选的真实性风险结果，调用 `interview_question_v1.md`，生成面试官可能追问的问题、提问原因、建议回答和风险提醒。

## 7. AI 提示词与 JSON 输出说明

项目中的 Prompt 都放在 `prompts/` 目录，没有散落在业务代码中。

每个 Prompt 都有明确的真实性约束，例如不能编造用户没有提供的工作经历、项目经历、学历、证书、公司、成果指标或技能经验。模型可以做的是重组、压缩、突出和润色已有内容。

为了让后端稳定解析，AI 输出尽量使用 JSON。例如 JD 分析会返回 `required_skills`、`bonus_skills`、`responsibilities`、`keywords` 等字段；匹配报告会返回 `score`、`strengths`、`weaknesses` 等字段；定制简历会返回 `markdown` 和 `change_explanations`。

如果 AI 返回非 JSON、空结果或网络错误，后端会给出清晰错误提示。测试代码中也通过 mock AI 的方式避免真实调用外部模型。

## 8. 数据库设计说明

数据库使用 SQLite，核心表包括：

- `users`：Demo 阶段保留默认用户。
- `resume_profiles`：保存通用简历。
- `projects`：保存项目库。
- `job_descriptions`：保存岗位 JD 原文。
- `job_analyses`：保存 JD 分析结果。
- `match_reports`：保存匹配度报告。
- `resume_versions`：保存生成的简历版本。
- `truth_check_results`：保存真实性风险检测结果。
- `interview_question_results`：保存面试追问预测结果。

设计上把用户原始资料和 AI 生成结果分开存储，这样可以追溯每个结果的来源，也便于后续扩展历史版本、审计和重新生成。

## 9. Demo 演示顺序

演示时建议按首页的 8 步流程进行：

1. 打开 Dashboard，展示整体流程和数据统计。
2. 进入通用简历页面，输入并保存简历。
3. 进入项目库页面，添加项目经历。
4. 进入岗位 JD 页面，输入 JD 并点击分析。
5. 进入匹配分析页面，选择简历、项目和 JD，生成匹配报告。
6. 进入简历版本页面，选择匹配报告，生成定制简历。
7. 在同一页面进行真实性风险检测。
8. 继续生成面试追问预测。
9. 点击导出 Markdown，展示生成的 `.md` 文件。

如果现场 AI 调用失败，可以说明原因通常是网络或 API Key 配置问题，并展示后端错误提示、Prompt 文件和 mock 测试结果，说明系统结构和异常处理是完整的。

## 10. 项目亮点总结

我认为项目主要有几个亮点：

第一，功能链路完整，从资料输入到导出文件形成闭环。

第二，项目结构比较清晰，后端没有把业务逻辑写在一个大文件中，而是按模块分层。

第三，AI 调用是供应商中立的，后续切换模型比较方便。

第四，项目没有只追求“生成更好看的简历”，而是加入了真实性风险检测和面试追问预测，更符合实际求职场景。

第五，AI 输出采用结构化 JSON，方便后端保存、前端展示和后续扩展。

## 11. 不足与改进方向

当前项目仍然是 Demo 阶段，也有一些不足：

- 暂时没有真实登录系统，Demo 使用默认用户。
- 数据库使用 SQLite，适合本地演示，但生产环境需要升级到 PostgreSQL。
- 暂不支持 PDF/Word 导出，只支持 Markdown。
- UI 还有进一步组件化和交互优化空间。
- AI 调用依赖外部 API，现场演示时需要保证网络和 API Key 配置。
- 暂不支持简历文件解析和招聘网站自动获取 JD。

后续可以扩展用户体系、云端同步、多模型选择、更多导出格式、模板系统和求职进度管理。

## 12. 结束语

总的来说，ResumeFit 完成了一个基于大语言模型的智能简历定制 Demo。项目重点是帮助用户基于真实经历理解岗位要求、改进简历表达、控制真实性风险，并准备面试追问。

通过这个项目，我实践了 Vue 3 前端开发、FastAPI 后端分层、SQLite 数据建模、OpenAI-compatible AI 调用、Prompt 设计和测试验证等内容。以上就是我的答辩介绍，谢谢各位老师。
