# ResumeFit V0.2 产品化体验版验收清单

- [ ] `/versions` 页面组件拆分

- [ ] 加载状态与防重复点击

- [ ] `/versions` 历史记录体验优化
- [ ] Dashboard 产品化引导

### 2. 正常完整流程测试

操作步骤：
1. 打开 `/resume`。
2. 输入简历标题，例如：`后端开发通用简历`。
3. 输入简历正文，例如包含教育背景、技能、项目经历概述的 Markdown 文本。
4. 点击保存。
5. 打开 `/projects`。
6. 添加一个项目，填写项目名称、项目类型、我的角色、技术栈、项目描述、个人贡献和作品链接。
7. 打开 `/jobs`。
8. 输入公司名称、岗位名称和 JD 原文。
9. 点击保存并分析 JD。
10. 打开 `/analysis`。
11. 选择简历、项目和已分析 JD。
12. 点击生成匹配报告。
13. 打开 `/versions`。
14. 选择匹配报告，点击生成定制简历。
15. 对生成的 ResumeVersion 执行真实性风险检测。
16. 执行面试追问预测。
17. 点击导出 Markdown。

预期结果：
- [ ] 简历保存成功，列表展示新简历
- [ ] 项目保存成功，列表展示新项目
- [ ] JD 保存并分析成功，展示结构化分析结果
- [ ] 匹配报告生成成功，展示分数、优势、不足、缺失关键词、修改建议和真实性提醒
- [ ] 定制简历生成成功，展示 Markdown 内容和修改原因
- [ ] 真实性风险检测生成成功，展示风险等级、风险表达、证据状态和更稳妥改法
- [ ] 面试追问预测生成成功，展示问题、原因、关联内容、建议回答、回答策略和风险提醒
- [ ] Markdown 文件成功下载
- [ ] Dashboard 刷新后进度变为 `5 / 5`

是否通过：
- [ ] 通过
- [ ] 不通过
