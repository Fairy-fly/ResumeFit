# ResumeFit V0.2 产品化体验版最终验收清单

本文档用于 ResumeFit V0.2 产品化体验版的最终手动验收。V0.2 只优化前端体验和演示可用性，不新增登录、支付、会员、爬取、PDF/Word 导出等商业化功能。

## 一、V0.2 已完成优化项

- [ ] 固定左侧 Sidebar，长页面滚动时导航不跟随右侧内容滚动
- [ ] `/resume`、`/projects`、`/jobs` 历史内容详情查看
- [ ] 历史内容折叠，长列表不再默认铺满页面
- [ ] `/versions` 页面组件拆分
- [ ] `/versions` 历史 ResumeVersion 选择后回填 Markdown 预览
- [ ] 复制 Markdown 和导出 Markdown 都作用于当前选中的 ResumeVersion
- [ ] `/versions` 生成上下文折叠
- [ ] `/versions` 修改原因卡片优化，重复标题能区分展示
- [ ] 空状态与下一步 CTA
- [ ] 加载状态与防重复点击
- [ ] 错误提示产品化
- [ ] 历史记录体验优化
- [ ] Dashboard 产品化引导

## 二、验收前准备

### 1. 启动后端

```powershell
cd D:\cxdownload\ResumeFit\backend
.\.venv\Scripts\Activate.ps1
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

- [ ] 后端启动成功
- [ ] 浏览器访问 `http://localhost:8000/docs` 可打开 FastAPI docs
- [ ] 浏览器访问 `http://localhost:8000/health` 返回正常

### 2. 启动前端

```powershell
cd D:\cxdownload\ResumeFit\frontend
npm run dev
```

- [ ] 前端启动成功
- [ ] 浏览器访问 `http://localhost:5173/` 可打开首页

### 3. AI 配置确认

- [ ] `backend/.env` 中已配置 `AI_API_KEY`
- [ ] `AI_BASE_URL` 和 `AI_MODEL` 使用 OpenAI-compatible 配置
- [ ] 没有把真实 API Key 写入代码、文档、测试文件或提交记录

### 2. `/resume` 通用简历页面

操作步骤：
1. 打开 `/resume`。
2. 如果没有数据，查看空状态提示。
3. 输入简历标题，例如 `后端开发通用简历`。
4. 输入简历正文，例如包含教育背景、技能和项目经历的 Markdown 文本。
5. 点击保存按钮。
6. 展开“已保存简历”折叠区。
7. 点击某条简历的“查看详情”。

预期结果：
- [ ] 空数据时显示“还没有通用简历”和下一步 CTA
- [ ] 简历保存成功后，已保存简历区域自动展开

- [ ] 详情弹窗展示完整标题、正文、创建时间和更新时间
- [ ] 详情弹窗可关闭，长正文可滚动

- [ ] 400 / 404 业务错误保留有价值的后端 detail
