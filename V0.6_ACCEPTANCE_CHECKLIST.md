# ResumeFit V0.6 管理后台基础版验收清单

本文档用于验收 ResumeFit V0.6。V0.6 只实现轻量管理后台、管理员识别、用户列表、用户详情、全站 AI 用量概览和账号启用/禁用，不包含支付、会员套餐、订单、发票、复杂 RBAC、操作审计、删除用户、修改密码、头像上传、招聘网站爬取或 PDF/Word 导出。

## 一、V0.6 已完成内容

- [ ] `users` 表新增 `role` 字段。
- [ ] 支持 `role="user"` 和 `role="admin"`。
- [ ] 轻量迁移可为旧 SQLite 数据库补齐 `role` 字段。
- [ ] `/auth/me`、注册、登录返回用户 `role`。
- [ ] 新增 admin 权限依赖，普通用户访问后台接口返回 `403`。
- [ ] 新增 `GET /admin/users`。
- [ ] 新增 `GET /admin/users/{user_id}`。
- [ ] 新增 `PATCH /admin/users/{user_id}/status`。
- [ ] 新增 `GET /admin/usage/summary`。
- [ ] 新增前端 `/admin` 管理后台页面。
- [ ] Sidebar 仅 admin 用户显示“管理后台”。
- [ ] 普通用户手动访问 `/admin` 时显示无权限提示。

## 二、管理员账号准备

V0.6 不提供公开“注册为管理员”入口，也不自动创建默认管理员账号。验收时建议：

1. 先通过注册页注册一个账号，例如 `admin@example.com`。
2. 在本地 SQLite 中手动设置为 admin。

示例：

```powershell
cd backend
.\.venv\Scripts\python.exe -c "from app.core.database import SessionLocal; from app.models.user import User; db=SessionLocal(); user=db.query(User).filter(User.email=='admin@example.com').first(); user.role='admin'; db.commit(); db.close()"
```

## 三、后端接口验收

### 1. 未登录访问后台接口

操作步骤：

1. 不携带 token 请求 `GET /admin/users`。
2. 不携带 token 请求 `GET /admin/usage/summary`。

预期结果：

- [ ] 返回 `401`。

### 2. 普通用户访问后台接口

操作步骤：

1. 注册普通用户。
2. 使用普通用户 token 请求：
   - `GET /admin/users`
   - `GET /admin/users/{user_id}`
   - `PATCH /admin/users/{user_id}/status`
   - `GET /admin/usage/summary`

预期结果：

- [ ] 均返回 `403`。

### 3. Admin 查看用户列表

操作步骤：

1. 使用 admin token 请求 `GET /admin/users?page=1&page_size=20`。
2. 使用 `search` 参数按 email 或昵称搜索。

预期结果：

- [ ] 返回用户列表。
- [ ] 返回 `total`、`page`、`page_size`。
- [ ] 用户项包含 `id`、`email`、`display_name`、`role`、`status`、`created_at`、`updated_at`、`usage`。
- [ ] 不包含 `password_hash`。
- [ ] 不包含 token。

### 4. Admin 查看用户详情

操作步骤：

1. 使用 admin token 请求 `GET /admin/users/{user_id}`。

预期结果：

- [ ] 返回用户基础信息。
- [ ] 返回 `usage_summary`。
- [ ] 返回最近 AI 调用记录。
- [ ] 不返回 `password_hash`。
- [ ] 不返回 token。
- [ ] 不返回用户简历正文、项目正文、JD 原文或 AI 生成内容。

### 5. Admin 查看全站用量

操作步骤：

1. 使用 admin token 请求 `GET /admin/usage/summary`。

预期结果：

- [ ] 返回 `total_call_count`。
- [ ] 返回 `monthly_call_count`。
- [ ] 返回 `monthly_success_count`。
- [ ] 返回 `monthly_failure_count`。
- [ ] 返回 `feature_counts`。
- [ ] 返回 `recent_calls`。

### 6. Admin 启用 / 禁用用户

操作步骤：

1. 使用 admin token 调用 `PATCH /admin/users/{user_id}/status`，设置普通用户为 `disabled`。
2. 使用该普通用户重新登录。
3. 使用该普通用户旧 token 调用任意业务接口。
4. 再使用 admin token 设置该用户为 `active`。

预期结果：

- [ ] 禁用接口成功返回 `disabled`。
- [ ] 被禁用用户不能登录。
- [ ] 被禁用用户旧 token 不能继续访问业务接口。
- [ ] 重新启用后用户可再次登录。
- [ ] Admin 不能禁用自己的账号。
