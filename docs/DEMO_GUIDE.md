# Demo Guide

本文档用于演示 ResumeFit 的完整 AI 简历优化流程。

## 启动后端

```powershell
cd ResumeFit
cd backend
.\.venv\Scripts\Activate.ps1
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

后端文档地址：

- `http://localhost:8000/docs`

## 启动前端

```powershell
cd ResumeFit
cd frontend
npm run dev
```

前端地址：

- `http://localhost:5173`

## 演示前准备

1. 复制 `.env.example` 为 `.env`。
2. 配置本地 `AI_API_KEY`。
3. 确认后端和前端都已启动。
4. 准备一份通用简历、一段项目经历和一个目标岗位 JD。

## 推荐演示顺序

### 1. 注册 / 登录

- 打开 `http://localhost:5173`。
- 注册一个测试账号或登录已有账号。
- 说明系统使用 JWT 鉴权，多用户数据互相隔离。

预期结果：

- 登录后进入 Dashboard。
- 左侧显示当前用户和功能导航。

### 2. Dashboard

- 展示首页流程、完成进度和下一步建议。
- 说明系统从简历、项目、JD 到定制简历和导出形成闭环。

预期结果：

- 页面展示完整操作路径。
- 快捷入口可以进入各功能页。

### 3. 通用简历

- 进入 `/resume`。
- 输入或粘贴基础简历。
- 保存后查看历史详情。

预期结果：

- 简历保存成功。
- 历史列表可查看完整正文。

### 4. 项目库

- 进入 `/projects`。
- 添加项目名称、角色、技术栈、项目描述、个人贡献和链接。

预期结果：

- 项目保存成功。
- 项目详情可展开查看。

### 5. 岗位 JD 分析

- 进入 `/jobs`。
- 粘贴目标岗位 JD。
- 点击保存并分析。

预期结果：

- 系统展示必备技能、加分技能、职责、关键词和简历侧重点。

### 6. 匹配度报告

- 进入 `/analysis`。
- 选择简历、项目和已分析 JD。
- 生成匹配度报告。

预期结果：

- 展示匹配分数、优势、差距和修改建议。

### 7. 定制简历版本

- 进入 `/versions`。
- 选择匹配报告并生成定制简历。
- 查看 Markdown 预览和修改原因。

预期结果：

- 生成岗位定制版本。
- 可以切换历史版本。

### 8. 导出

- 在 `/versions` 点击复制 Markdown、导出 Markdown 或导出 DOCX。
- DOCX 可选择 standard、modern、compact 模板。

预期结果：

- 下载文件属于当前选中的简历版本。
- 其他用户不能导出该版本。

### 9. 真实性风险检测

- 在 `/versions` 对当前版本运行真实性风险检测。

预期结果：

- 展示风险总结、风险项和保守表达建议。

### 10. 面试追问预测

- 在 `/versions` 对当前版本生成面试追问。

预期结果：

- 展示问题、追问原因、关联内容、难度、建议回答、策略和风险提醒。

### 11. 用量统计

- 进入 `/usage`。
- 查看本月额度、剩余额度、功能调用分布和最近调用记录。

预期结果：

- AI 调用记录按当前用户统计。

### 12. 个人中心

- 进入 `/account`。
- 查看账号信息、用量概览，并修改昵称。

预期结果：

- 修改昵称后侧边栏同步更新。

## AI 调用失败时如何说明

如果 AI 调用失败，可以按以下方式解释：

- 本地 `.env` 需要配置 `AI_API_KEY`。
- 网络或模型服务异常时，前端会显示可读错误。
- 测试环境中 AIClient 使用 mock，因此自动化测试不依赖真实 DeepSeek 调用。
