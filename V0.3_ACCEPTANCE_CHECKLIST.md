| 用户 | 邮箱 | 密码建议 |

| 用户 B | `userB@example.com` | `password123` |

## 三、手动验收步骤

### 1. 注册用户 A

操作步骤：

- 打开前端页面。
- 进入 `/register`。
- 输入：
  - email：`userA@example.com`
  - display_name：`User A`
  - password：`password123`
  - confirm_password：`password123`
- 点击注册按钮。

预期结果：

- [ ] 注册成功后进入 Dashboard。
- [ ] 侧边栏显示当前用户 `User A` 或 `userA@example.com`。
- [ ] 刷新页面后仍保持登录状态。

### 2. 用户 A 创建简历、项目、JD

操作步骤：

- 进入 `/resume`，填写一份通用简历并保存。
- 进入 `/projects`，添加一个项目经历并保存。
- 进入 `/jobs`，填写公司名称、岗位名称、JD 原文，并点击保存/分析。

预期结果：

- [ ] `/resume` 可以看到用户 A 的简历。
- [ ] `/projects` 可以看到用户 A 的项目。
- [ ] `/jobs` 可以看到用户 A 的 JD。
- [ ] JD 分析成功后能看到分析结果。

### 3. 用户 A 生成匹配报告

操作步骤：

- 进入 `/analysis`。
- 选择用户 A 的简历、项目和已分析 JD。
- 点击生成匹配报告。

预期结果：

- [ ] 成功生成匹配报告。
- [ ] 页面显示匹配分数、优势、不足、缺失关键词、修改建议和真实性提醒。

### 4. 用户 A 生成定制简历

操作步骤：

- 进入 `/versions`。
- 选择用户 A 的匹配报告。
- 点击生成定制简历。

预期结果：

- [ ] 成功生成 ResumeVersion。

## 四、接口级验收

以下接口级验收可以使用 Swagger 或 PowerShell。Swagger 地址为：

```text
http://localhost:8000/docs
```

### 1. 获取 token

PowerShell 示例：

```powershell
$userA = Invoke-RestMethod http://localhost:8000/auth/login `
  -Method Post `
  -ContentType "application/json" `
  -Body '{"email":"userA@example.com","password":"password123"}'

$userB = Invoke-RestMethod http://localhost:8000/auth/login `
  -Method Post `
  -ContentType "application/json" `
  -Body '{"email":"userB@example.com","password":"password123"}'

$headersA = @{ Authorization = "Bearer $($userA.access_token)" }
$headersB = @{ Authorization = "Bearer $($userB.access_token)" }
```

### 2. 无 token 访问业务接口返回 401

操作步骤：

